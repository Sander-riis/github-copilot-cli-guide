# Phase 1 — UI Design Contract

> Visual and interaction contract for the HTML shell phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Sources: CONTEXT.md (D-01–D-07), STACK.md (palette, typography, spacing), ARCHITECTURE.md (component patterns, CSS tokens).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — bespoke CSS with CSS Custom Properties |
| Preset | not applicable |
| Component library | none — vanilla HTML + CSS |
| Icon library | inline SVG only (no CDN) |
| Font | System UI sans + system monospace (no web font requests) |

**Rationale (from STACK.md):** No CSS framework — Bootstrap/Tailwind both outweigh the entire page content. CSS Custom Properties replace preprocessor variables with zero tooling.

## Spacing Scale

All values are multiples of 4 (8-point sub-grid). Source: ARCHITECTURE.md `--spacing-*` tokens.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-text gaps, inline badge padding |
| sm | 8px | Tab button vertical padding, sidebar link padding, copy button padding |
| md | 16px | Section body horizontal padding, collapsible body indent |
| lg | 24px | Tab button horizontal padding, callout box padding, sidebar section gap |
| xl | 32px | Content section vertical margin between `<section>` blocks |
| 2xl | 48px | Major inter-topic breaks (between `<article>` sibling sections within a tab panel) |

Exceptions:
- Header height: **56px** (fixed, ARCHITECTURE.md `--header-height`)
- Sidebar width: **260px** (fixed, ARCHITECTURE.md `--sidebar-width`)
- Content max-width: **860px** (fixed, ARCHITECTURE.md `--content-max-width`)
- Touch target minimum: **44px tall** for tab buttons and copy buttons
- Border radius: **6px** (components), **12px** (code block wrappers) — ARCHITECTURE.md `--border-radius`

## Typography

Two font families, four size steps. Source: ARCHITECTURE.md `--font-*` tokens.

### Font Families

```css
/* UI / prose / labels */
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Helvetica, Arial, sans-serif;

/* Code, paths, inline code, terminal snippets */
font-family: "SFMono-Regular", Consolas, "Liberation Mono", Menlo, Courier, monospace;
```

### Type Scale

| Role | Size | Weight | Line Height | Font Family | Usage |
|------|------|--------|-------------|-------------|-------|
| Body | 15px | 400 (regular) | 1.6 | sans | All prose paragraph text |
| Label/Caption | 13px | 400 (regular) | 1.4 | sans | Sidebar link text, copy button text, `<summary>` secondary text, muted meta |
| Heading (h2/h3) | 17px | 600 (semibold) | 1.3 | sans | Section headings (`<h2>` in content panels), sidebar group labels, callout type label, collapsible `<summary>` primary text |
| Display (h1) | 20px | 600 (semibold) | 1.2 | sans | Page header title only — "GitHub Copilot CLI" |
| Code | 13px | 400 (regular) | 1.5 | mono | All `<code>` and `<pre>` content, language badge, file path label |

**Rules:**
- Maximum 2 weights in use: 400 (regular) and 600 (semibold). Never use 500, 700, or bold.
- Heading line-height 1.3, body line-height 1.6, code 1.5 — no exceptions.
- Inline `<code>` in prose uses 13px mono, with `--color-bg-overlay` background and 4px horizontal padding.

## Color

Full palette from GitHub's Primer dark scale. Source: ARCHITECTURE.md `:root` block + STACK.md color table.

```css

--color-bg-canvas:     #0d1117;   /* full-page background */

--color-bg-overlay:    #1c2128;   /* code block backgrounds */
--color-bg-subtle:     #21262d;   /* hover states, striped rows */

/* Borders */
--color-border:        #30363d;
--color-border-muted:  #21262d;
--color-border-subtle: #484f58;   /* active/focus states */

/* Text */
--color-text-primary:  #e6edf3;
--color-text-secondary:#8b949e;
--color-text-link:     #58a6ff;

/* Accent */
--color-accent:        #3fb950;   /* terminal green */

--color-success:       #3fb950;
```

### 60/30/10 Distribution

| Role | Hex | CSS Token | Usage |
|------|-----|-----------|-------|
| Dominant (60%) | `#0d1117` | `--color-bg-canvas` | Full-page background, all undecorated surfaces |
| Secondary (30%) | `#161b22` | `--color-bg-surface` | Header bar, sidebar, card backgrounds, tab panel wrappers |
| Accent (10%) | `#3fb950` | `--color-accent` | Active tab underline bar, copy-button success state ("Copied!") text + icon, scrollspy active indicator bar |

**Accent reserved for (explicit list):**
1. Active tab bottom border (2px underline on the selected tab button)
2. Scrollspy active sidebar link left indicator (3px left border on `sidebar-nav__link--active`)
3. Copy button label transition — text color + icon color change to accent on "Copied!" state

**Accent is NOT used for:** hover states, body links (use `--color-text-link: #58a6ff`), headings, or general decoration.

### Elevated Surface (code blocks)

| Token | Hex | Usage |
|-------|-----|-------|
| `--color-bg-overlay` | `#1c2128` | `<pre>` code block background |
| `--color-bg-subtle` | `#21262d` | Hover state on tab buttons, sidebar links, copy button |

### Semantic Colors

Used exclusively for callout boxes (left border + subtle background tint). **Not used anywhere else.**

| Variant | Border Color | Background Tint | Text Color |
|---------|-------------|-----------------|------------|

Destructive: `#f85149` (`--color-danger`) — reserved for `callout--danger` boxes only. **No destructive actions exist in this phase** (static reference guide, no server calls, no data mutation).

## Component Specifications

### 1. Page Header

```
┌─────────────────────────────────────────────────────────┐
│ [Copilot SVG icon]  GitHub Copilot CLI  Agents & Skills  │  ← 56px tall, --color-bg-surface bg
│                     [CLI Intro] [Agents] [Skills]         │  ← tabs flush to bottom of header
└─────────────────────────────────────────────────────────┘
```

- Height: **56px** fixed
- Background: `--color-bg-surface` (`#161b22`)
- Bottom border: 1px solid `--color-border`
- Title: "GitHub Copilot CLI" at 20px/600, `--color-text-primary`
- Subtitle: "Agents & Skills Guide" at 13px/400, `--color-text-secondary`, same line
- Copilot icon: inline SVG, 20×20px, `--color-text-secondary` fill

### 2. Tab Navigation

Three tabs: **CLI Intro** | **Agents** | **Skills**

**Default state:**
- Padding: 8px top/bottom × 24px left/right (meets 44px tall touch target)
- Background: transparent
- Color: `--color-text-secondary` (`#8b949e`)
- Border-bottom: 2px solid transparent
- Font: 15px/400
- Transition: `color 0.15s ease, border-color 0.15s ease`

**Hover state:**
- Color: `--color-text-primary`
- Background: `--color-bg-subtle` (`#21262d`)

**Active state:**
- Color: `--color-text-primary`
- Border-bottom: 2px solid `--color-accent` (`#3fb950`)
- Font weight: 600

**Tab switch behavior:** CSS opacity fade — `opacity 0` to `opacity 1` over **150ms ease-in** on the incoming panel. Outgoing panel hides instantly (display:none after transition). No slide or translate animations — cleaner for a documentation tool.

**ARIA:** `role="tablist"` on nav, `role="tab"` on each button, `aria-selected="true/false"`, `aria-controls="section-{tab}"`.

### 3. Layout Grid

```
body {
  display: grid;
  grid-template-rows: 56px 1fr;
  grid-template-areas: "header header" "sidebar main";
  grid-template-columns: 260px 1fr;
  height: 100vh;
  overflow: hidden;
}
```

- Sidebar: 260px, `overflow-y: auto`, `--color-bg-surface` background
- Main content: fluid, `overflow-y: auto`, max-width 860px centered within the column
- Both sidebar and main scroll independently

### 4. Sidebar Navigation

Three pre-authored `<nav>` elements (one per tab), shown/hidden with tabs.

**Section group label:**
- Font: 13px/600, `--color-text-secondary`, letter-spacing 0.08em, UPPERCASE
- Padding: 16px left, 8px top
- Not a link

**Navigation link (default):**
- Font: 13px/400, `--color-text-secondary`
- Padding: 4px 8px 4px 16px
- No border
- Hover: color → `--color-text-primary`, background → `--color-bg-subtle`
- Transition: `color 0.1s, background 0.1s`

**Active scrollspy link:**
- Color: `--color-text-link` (`#58a6ff`)
- Box-shadow: `inset 3px 0 0 var(--color-accent)` (replaces border-left — does not affect layout)
- Padding-left: 16px (standard md scale value; box-shadow does not consume layout space)
- Font weight: 600

**Scrollspy mechanism:** `IntersectionObserver` with `threshold: 0.4` on each `<section>` within the active article. Fires `sidebar-nav__link--active` class on the corresponding anchor link.

### 5. Code Block Component

```html
<div class="code-block" data-lang="bash">
  <div class="code-block__header">
    <span class="code-block__lang">bash</span>
    <!-- optional: <span class="code-block__filepath">.github/agents/my-agent.md</span> -->
    <button class="code-block__copy" aria-label="Copy to clipboard">
      [clipboard SVG] <span class="code-block__copy-label">Copy</span>
    </button>
  </div>
  <pre class="code-block__pre"><code class="language-bash">...</code></pre>
</div>
```

**Header bar:**
- Height: 36px
- Background: `--color-bg-subtle` (`#21262d`)
- Border-bottom: 1px solid `--color-border`
- Border-radius: 6px 6px 0 0

**Language badge:**
- Font: 13px/400 mono, `--color-text-secondary`
- Uppercase
- Left-aligned with 12px padding

**File path variant** (`.code-block--file`):
- Replace lang badge with file icon SVG (12×12) + filepath text in 13px mono `--color-text-secondary`

**Copy button (default):**
- Position: absolute top-right, 8px offset
- Background: transparent
- Border: 1px solid `--color-border`
- Border-radius: 6px
- Padding: 4px 8px
- Font: 13px/400, `--color-text-secondary`
- Icon: clipboard SVG 14×14, `--color-text-secondary` fill
- Label: "Copy code" (text visible alongside icon)
- Hover: border-color → `--color-border-subtle`, color → `--color-text-primary`
- Transition: `border-color 0.15s, color 0.15s`

**Copy button (success state — 2 seconds):**
- Icon: checkmark SVG 14×14, `--color-accent` fill
- Label: "Copied!"
- Color: `--color-accent` (`#3fb950`)
- Border-color: `--color-accent`
- Returns to default after 2000ms

**Code body:**
- Background: `--color-bg-overlay` (`#1c2128`)
- Padding: 16px
- Border-radius: 0 0 6px 6px
- Border: 1px solid `--color-border` (top excluded — header provides visual top)

- Overflow-x: auto (horizontal scroll for long lines, never wraps)
- Tab size: 2

### 6. Collapsible Component

Uses native `<details>`/`<summary>`. Zero JavaScript required.

```html
<details class="collapsible">
  <summary class="collapsible__trigger">
    <svg class="collapsible__chevron"><!-- right-pointing chevron --></svg>
    Section Title
  </summary>
  <div class="collapsible__body">...</div>
</details>
```

**Summary (trigger):**
- Font: 15px/600, `--color-text-primary`
- Padding: 12px 16px
- Background: `--color-bg-surface`
- Border-radius: 6px (collapses to 6px 6px 0 0 when open)
- Border: 1px solid `--color-border`
- Cursor: pointer
- `list-style: none` — hide browser default triangle
- Hover: background → `--color-bg-subtle`

**Chevron icon:**
- 16×16px inline SVG, `--color-text-secondary` fill
- Default: pointing right (→)
- Open state: rotate 90° (→ becomes ↓)
- Transition: `transform 0.2s ease`
- CSS: `details[open] .collapsible__chevron { transform: rotate(90deg); }`

**Body:**
- Padding: 16px
- Background: `--color-bg-canvas`
- Border: 1px solid `--color-border`, border-top: none
- Border-radius: 0 0 6px 6px

## Interaction Contracts

### Tab Switching
- Click on inactive tab → activate that tab + deactivate others (update `aria-selected`, swap `content-panel--active/hidden` classes, swap `sidebar-nav--hidden` classes)
- Scroll position resets to 0 for both sidebar and main content on tab switch
- Default active tab on load: **CLI Intro** (first tab)
- No animated transition on load — only on subsequent switches

### Scrollspy
- `IntersectionObserver` with `{ threshold: 0.4, rootMargin: "-56px 0px 0px 0px" }` (accounts for sticky header)
- As a `<section>` reaches 40% visibility, its corresponding sidebar link becomes active
- Only one sidebar link active at a time — deactivate all others when a new section enters
- Observer only active for the currently visible tab panel (disconnect/reconnect on tab switch)

### Copy to Clipboard
- `navigator.clipboard.writeText(code)` — async
- On success: button transitions to "Copied!" + checkmark icon for exactly **2000ms**, then reverts
- On failure: silent fail — button returns to "Copy" immediately (no error display)
- Copy extracts `innerText` from the `<code>` element (stripped of HTML, preserves whitespace)

### Collapsibles
- Native `<details>` open/close — no JavaScript
- Chevron rotation via CSS `details[open]` selector
- Multiple collapsibles can be open simultaneously (no accordion behavior)

## Accessibility Baseline

| Requirement | Implementation |
|-------------|---------------|
| Tab ARIA roles | `role="tablist"`, `role="tab"`, `aria-selected`, `aria-controls` |
| Tab panel roles | `role="tabpanel"`, `aria-labelledby` pointing to tab button ID |
| Copy button | `aria-label="Copy to clipboard"` — text label also visible |
| Collapsible | Native `<details>`/`<summary>` — screen reader accessible by default |
| Color contrast | All text on dark backgrounds meets WCAG AA (4.5:1 minimum) — GitHub Primer palette is pre-tested |
| Focus indicators | Browser default focus ring, enhanced: `outline: 2px solid --color-text-link` on `:focus-visible` |
| Keyboard navigation | Tab to focus buttons, Enter/Space to activate — all via native HTML |
| `lang` attribute | `<html lang="en">` |
