          <h3 style="margin: 32px 0 8px;">Subagent Orchestration</h3>
          <p>An agent becomes an orchestrator when it invokes other agents as subagents. The orchestrating agent receives the full task, decomposes it, assigns each piece to the appropriate subagent, dispatches them, and synthesizes the results. Each subagent runs in its own isolated context window — it does not share memory with the orchestrator or with other subagents running simultaneously.</p>
          <p>Context isolation is the architectural property that makes parallelism safe. A <code>code-review</code> subagent and a <code>task</code> subagent can both inspect the same codebase simultaneously without their findings interfering. The orchestrator sees only the subagent outputs, not the intermediate reasoning each accumulated. This also means subagents cannot accidentally leak sensitive context between tasks.</p>
