CLI Overview

GitHub Copilot CLI runs as a terminal-native AI assistant. It reads your codebase, executes commands, and makes changes — all through natural language.

The agent operates in your working directory with access to tools. Every action goes through the permission system.

Tool Access

Four core tools power the agent: file system (view, edit, create, glob, grep), shell execution (bash/powershell), sub-agents (task delegation), and web fetch.

Permission modes control what Copilot can do without asking: suggest (read-only), auto-edit (file changes without confirmation), and yolo (full autonomy).

Environment & Config

Configuration lives in .github/copilot-instructions.md. Custom instructions shape coding style, tools, and architecture preferences.

Best Practices

Start broad, then narrow. Give context about goals before specific changes. Use slash commands to manage sessions.

Agents

Five built-in agents handle exploration, testing, code review, general tasks, and plan validation. Custom agents extend this with domain knowledge.

Skills

Skills teach domain knowledge. They augment the main agent rather than running autonomously. Trigger by keyword or explicit invocation.

MCP Servers

Model Context Protocol servers connect Copilot to external services — GitHub APIs, databases, web fetching, and custom integrations.
